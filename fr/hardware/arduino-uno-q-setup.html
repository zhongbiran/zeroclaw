ZeroClaw sur Arduino Q — Guide étape par étape

Exécutez ZeroClaw sur la partie Linux de l’Arduino Uno Q. Telegram fonctionne via WiFi ; le contrôle des GPIO utilise le pont (nécessite une application minimale d’App Lab).

Ce qui est inclus (aucun changement de code nécessaire)

ZeroClaw inclut tout ce qui est nécessaire pour Arduino Uno Q. Clonez le dépôt et suivez ce guide — aucun patch ni code personnalisé requis.

ComposantEmplacementObjectif
Application de pontfirmware/uno-q-bridge/Croquis MCU + serveur socket Python (port 9999) pour GPIO
Outils de pontcrates/zeroclaw-hardware/src/peripherals/uno_q_bridge.rsOutils gpio_read / gpio_write qui communiquent avec le pont via TCP
Commande de configurationcrates/zeroclaw-hardware/src/peripherals/uno_q_setup.rszeroclaw peripheral setup-uno-q déploie le pont via scp + arduino-app-cli
Schéma de configurationboard = "arduino-uno-q", transport = "bridge"Pris en charge dans config.toml

Compilez avec --features hardware pour inclure la prise en charge d’Uno Q.

Prérequis

  • Arduino Uno Q avec WiFi configuré
  • Arduino App Lab installed on your computer (for initial board setup)
  • arduino-app-cli available on the Uno Q (pre-installed with the board’s Debian image, used for Bridge deployment)
  • Clé API pour le LLM (OpenRouter, etc.)

Phase 1 : Configuration initiale d’Uno Q (à faire une seule fois)

1.1 Configurer Uno Q via App Lab

  1. Téléchargez Arduino App Lab (tar.gz sur Linux).
  2. Connectez le Uno Q via USB, puis mettez-le sous tension.
  3. Ouvrez App Lab, connectez-vous à la carte.
  4. Suivez l’assistant d’installation :
    • Définir le nom d’utilisateur et le mot de passe (pour SSH)
    • Configurer le WiFi (SSID, mot de passe)
    • Appliquez toutes les mises à jour du micrologiciel
  5. Notez l’adresse IP affichée (par exemple arduino@192.168.1.42) ou trouvez-la plus tard via ip addr show dans le terminal d’App Lab.

1.2 Vérifier l’accès SSH

ssh arduino@<UNO_Q_IP>
# Enter the password you set

Phase 2 : Installer ZeroClaw sur Uno Q

Option A : Compiler sur l’appareil (Plus simple, ~20–40 min)

# SSH vers Uno Q
ssh arduino@<UNO_Q_IP>

# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source ~/.cargo/env

# Installer les dépendances de construction (Debian)
sudo apt-get update
sudo apt-get install -y pkg-config libssl-dev

# Clonez zeroclaw (ou copiez votre projet via scp)
git clone https://github.com/zeroclaw-labs/zeroclaw.git
cd zeroclaw

# Build (prend environ 15–30 min sur Uno Q)
export CARGO_BUILD_JOBS=1 # build will be OOM-killed mid-link without this
cargo build --release --features hardware

# Installer
sudo cp target/release/zeroclaw /usr/local/bin/

Option B : Compilation croisée sur Mac (plus rapide)

# Sur votre Mac — ajoutez la cible aarch64
rustup target add aarch64-unknown-linux-gnu

# Installer le compilateur croisé (macOS ; requis pour l’édition des liens)
brew tap messense/macos-cross-toolchains
brew install aarch64-unknown-linux-gnu

# Construire
CC_aarch64_unknown_linux_gnu=aarch64-unknown-linux-gnu-gcc cargo build --release --target aarch64-unknown-linux-gnu --features hardware

# Copier vers Uno Q
scp target/aarch64-unknown-linux-gnu/release/zeroclaw arduino@<UNO_Q_IP>:~/
ssh arduino@<UNO_Q_IP> `sudo mv ~/zeroclaw /usr/local/bin/`

Si la compilation croisée échoue, utilisez l’option A et compilez sur l’appareil.

Phase 3 : Configurer ZeroClaw

3.1 Exécuter Onboard (ou créer la configuration manuellement)

ssh arduino@<UNO_Q_IP>

# Quick config
zeroclaw onboard --api-key YOUR_OPENROUTER_KEY --provider openrouter

# Ou créer la configuration manuellement
mkdir -p ~/.zeroclaw/workspace
nano ~/.zeroclaw/config.toml

3.2 Configuration minimale

Au minimum, définissez api_key / default_provider / default_model, ainsi que [channels.telegram] avec votre bot_token. Laissez [peripherals] désactivé jusqu’à la phase 4 ci-dessous. Consultez la référence de configuration pour l’ensemble des champs.

Phase 4 : Exécuter le démon ZeroClaw

ssh arduino@<UNO_Q_IP>

# Run daemon (Telegram polling works over WiFi)
zeroclaw daemon --host 127.0.0.1 --port 42617

À ce stade : Le chat Telegram fonctionne. Envoyez des messages à votre bot — ZeroClaw répond. Pas encore de GPIO.

Phase 5 : GPIO via pont (ZeroClaw s’en charge)

ZeroClaw inclut l’application Bridge et la commande de configuration.

5.1 Déployer l’application Bridge

From your computer (with zeroclaw repo):

zeroclaw peripheral setup-uno-q --host 192.168.0.48

Depuis Uno Q (connecté via SSH) :

zeroclaw peripheral setup-uno-q

Cela copie l’application Bridge vers ~/ArduinoApps/uno-q-bridge et la démarre.

5.2 Ajouter à la configuration

Activez [peripherals] et ajoutez une entrée [[peripherals.boards]] avec board = "arduino-uno-q" et transport = "bridge".

5.3 Exécuter ZeroClaw

zeroclaw daemon --host 127.0.0.1 --port 42617

Maintenant, lorsque vous envoyez un message à votre bot Telegram « Allumer la LED » ou « Définir la broche 13 à l’état haut », ZeroClaw utilise gpio_write via le pont.

Dépannage

  • « commande introuvable : zeroclaw » — Utilisez le chemin complet : /usr/local/bin/zeroclaw ou assurez-vous que ~/.cargo/bin est dans le PATH.
  • Telegram ne répond pas — Vérifiez le bot_token, les allowed_users, et que l’Uno Q a une connexion Internet (WiFi).
  • Manque de mémoire — Gardez les fonctionnalités minimales (--features hardware pour Uno Q) ; envisagez compact_context = true.
  • Les commandes GPIO sont ignorées — Assurez-vous que l’application Bridge est en cours d’exécution (zeroclaw peripheral setup-uno-q la déploie et la démarre). La configuration doit contenir board = "arduino-uno-q" et transport = "bridge".
  • Fournisseur LLM (GLM/Zhipu) — Utilisez default_provider = "glm" ou "zhipu" avec GLM_API_KEY dans l’environnement ou la configuration. ZeroClaw utilise le point de terminaison v4 approprié.
