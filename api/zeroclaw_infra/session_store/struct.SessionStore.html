fn list_sessions_with_metadata(&self) -> Vec<SessionMetadata>

Override the trait default so JSONL-backed channel hydration picks the most-recent sessions when truncating to MAX_CONVERSATION_SENDERS. The trait default stamps every key with Utc::now(), which makes the orchestrator’s sort_by_key(|m| Reverse(m.last_activity)) arbitrary once more than that many sessions are persisted.

Update the content of the last message in a session. Used for incremental persistence of streaming responses — append a placeholder first, then update_last periodically as more content arrives. Returns false if the session is empty. Default implementation is remove_last + append (backends can override for efficiency).
