1//! Verifiable Intent (VI) — Rust-native implementation of the VI specification.
2//!
3//! This module provides full lifecycle support for the Verifiable Intent layered
4//! credential system: issuance of L2/L3 credentials, chain verification, and
5//! constraint evaluation for commerce-gated agent actions.
6//!
7//! # Attribution
8//!
9//! This implementation is based on the Verifiable Intent open specification and
10//! reference implementation by agent-intent, available at
11//! <https://github.com/agent-intent/verifiable-intent>, licensed under the
12//! Apache License, Version 2.0. This Rust-native reimplementation follows the
13//! VI specification design (SD-JWT layered credentials, constraint model,
14//! three-layer chain) without copying source code from the reference
15//! implementation.
16//!
17//! # Architecture
18//!
19//! - [`types`] — Core data models (credentials, mandates, constraints, keys).
20//! - [`crypto`] — SD-JWT / KB-SD-JWT construction and verification primitives.
21//! - [`verification`] — Chain verification, constraint checking, binding integrity.
22//! - [`issuance`] — L2/L3 credential construction.
23//! - [`error`] — Machine-readable error taxonomy for policy decisions.
24//!
25//! # Extension
26//!
27//! This module is an internal subsystem. Integration into the tool execution
28//! surface is handled by the tool layer (see `src/tools/`). Config schema
29//! entries live in `src/config/schema.rs`.
30
