Load workspace identity files and build a system prompt.

Follows the OpenClaw framework structure by default:

  1. Tooling — tool list + descriptions
  2. Safety — guardrail reminder
  3. Skills — full skill instructions and tool metadata
  4. Workspace — working directory
  5. Bootstrap files — AGENTS, SOUL, TOOLS, IDENTITY, USER, BOOTSTRAP, MEMORY
  6. Date & Time — timezone for cache stability
  7. Runtime — host, OS, model

When identity_config is set to AIEOS format, the bootstrap files section is replaced with the AIEOS identity data loaded from file or inline JSON.

Daily memory files (memory/*.md) are NOT injected — they are accessed on-demand via memory_recall / memory_search tools.
