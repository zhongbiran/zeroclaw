Expand description

For a #[nested] HashMap<String, T> field, parse a get_prop/set_prop path of the form <my_prefix>.<field_name>.<hm_key>.<inner_suffix> and return the HashMap key + the fully-qualified inner name that the value type’s own get_prop / set_prop expects.

HashMap keys are user-controlled and may contain dots, URLs, or hostnames (for example providers.models.custom:https://example.invalid/v1.api-key). Current map-keyed config sections expose leaf fields, so split from the right and preserve any dots inside the runtime key.

Returns None when the path doesn’t match, letting the derive’s generated code fall through to the next nested field.
