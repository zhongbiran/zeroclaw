Expand description

Stable error code consumed by HTTP / CLI / dashboard. Add codes here as new failure cases land — never invent codes ad-hoc at call sites.

Variants§

§

PathNotFound

The supplied property path is not defined in the schema.

§

ValidationFailed

Generic schema validation failure (catch-all wrapping Config::validate() bails). Specific failure modes (provider_fallback_dangling, etc.) get their own codes as the validator is incrementally restructured.

§

ProviderFallbackDangling

providers.fallback references a key not present in providers.models.

§

ConfigChangedExternally

On-disk config differs from in-memory state (an out-of-band file edit happened despite the daemon-running rule). Caller should reload.

§

ReloadFailed

The daemon-reload step after a successful save failed; on-disk config has been reverted to the pre-write snapshot to keep state consistent.

§

OpNotSupported

JSON Patch operation type is not supported in this PR (move / copy).

§

SecretTestForbidden

JSON Patch test operation targeted a secret or derived-from-secret path; rejected to prevent differential value inference.

§

ValueTypeMismatch

The supplied JSON value does not match the field’s declared type (e.g. an array passed where a scalar was expected, or a non-string element in a Vec<String>).

§

RequiredFieldEmpty

A required scalar field was empty / missing / blank. Path identifies which one (e.g. gateway.host, tunnel.openvpn.config_file).

§

InvalidNumericRange

A numeric field was out of its allowed range (zero, negative, or above an upper bound). Path identifies which one.

§

InvalidFormat

A string did not match its allowed format — invalid URL, bad scheme, invalid path prefix, characters outside the allowed set.

§

InvalidEnumVariant

An enum / discriminator field carried a value not in the allowed set (e.g. tunnel.provider with an unknown name).

§

DanglingReference

A reference to another config entry pointed at something that doesn’t exist (e.g. agents.<x>.delegate_to naming a missing agent). Distinct from ProviderFallbackDangling — that’s the specific case the dashboard surfaces for the providers section.

§

InternalError

Catch-all server failure not classified above. Avoid in code; log the original error and convert to a more specific code where possible.
