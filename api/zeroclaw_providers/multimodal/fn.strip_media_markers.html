pub fn strip_media_markers(text: &str) -> String
Expand description

Replace media markers ([IMAGE:...], [PHOTO:...], [DOCUMENT:...], [FILE:...], [VIDEO:...], [VOICE:...], [AUDIO:...]) with [media attachment]. Match is case-insensitive to align with the channel attachment parsers, which all uppercase the kind before comparing (crates/zeroclaw-channels/src/util.rs::ATTACHMENT_KINDS, telegram.rs, discord.rs, qq.rs, whatsapp_web.rs).

Use before passing user-facing text to auxiliary chat_with_system calls (intent classification, summarization, delegation) so that local file paths from inbound channels do not leak to the upstream provider — the upstream API would otherwise receive a filesystem path as image_url.url and reject the request.

Auxiliary calls do not need to see the media content; they only route or summarize, so the placeholder is sufficient. The main agent loop continues to call prepare_messages_for_provider for full normalization.
