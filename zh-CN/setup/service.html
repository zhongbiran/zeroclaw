ZeroClaw Docs

服务管理

ZeroClaw 提供对 systemd（Linux）、launchctl（macOS）以及任务计划程序/Windows 服务（Windows）的一流服务集成支持。这三种方式均通过同一个 CLI 接口驱动：

zeroclaw service install     # 注册服务
zeroclaw service start       # 启动它
zeroclaw service stop        停止
zeroclaw service restart     # 停止 + 启动
zeroclaw service status      # 运行中 / 已停止，上次退出代码
zeroclaw service uninstall   # 删除它

平台特定的后端实现在 crates/zeroclaw-runtime/src/service/ 中。你无需深入了解它们——但了解它们生成的内容有助于调试。

Linux — systemd

zeroclaw service install 会在 ~/.config/systemd/user/zeroclaw.service 写入一个用户作用域的单元文件。

单位：

  • Type=simple，代理进程保持在前台
  • User= 设置为调用者用户
  • SupplementaryGroups=gpio spi i2c（如果硬件功能已编译，则启用）
  • Restart=on-failure 并带有 10 秒的退避时间
  • ExecStart=/home/$USER/.cargo/bin/zeroclaw daemon

手动控制

systemctl --user start zeroclaw
systemctl --user stop zeroclaw
systemctl --user status zeroclaw
systemctl --user enable zeroclaw     # 登录时启动

日志

journalctl --user -u zeroclaw -f        # follow
journalctl --user -u zeroclaw --since 1小时前

系统作用域（根）服务

如果您需要在用户登录之前启动 ZeroClaw（例如在无头 SBC 或 VPS 上），请以 root 用户身份运行安装命令：

sudo zeroclaw service install
sudo systemctl enable --now zeroclaw

当以 sudo/root 权限调用时，zeroclaw service install 会在 /etc/systemd/system/zeroclaw.service 创建一个系统作用域的单元文件，并配置一个专用的 zeroclaw 服务用户。

Linux — OpenRC

当存在 /run/openrc 时自动检测（Alpine、某些 Gentoo 配置）。

zeroclaw service install   # 写入 /etc/init.d/zeroclaw
rc-service zeroclaw start
rc-update add zeroclaw default    # 开机启动

macOS — LaunchAgent

zeroclaw service install 会写入 ~/Library/LaunchAgents/com.zeroclaw.daemon.plist 并加载它。

launchctl list | grep zeroclaw
launchctl unload ~/Library/LaunchAgents/com.zeroclaw.daemon.plist
launchctl load ~/Library/LaunchAgents/com.zeroclaw.daemon.plist

日志输出到 ~/Library/Logs/ZeroClaw/zeroclaw.log（标准输出）和 zeroclaw.err（标准错误）。

由 Homebrew 管理

如果通过 Homebrew 安装，brew services 是首选的接口：

brew services start zeroclaw
brew services restart zeroclaw
brew services info zeroclaw

不要将 zeroclaw service 的 CLI 命令与 brew services 混用——选择其中一种即可。两者都会写入 plist 文件，同时存在会混淆 launchctl。

Windows — 任务计划程序

zeroclaw service install 会在当前用户会话中创建一个计划任务：

  • 触发器：在登录时
  • 条件：电池、空闲和电源保存条件均已禁用（否则任务会意外停止）
  • 操作：以隐藏模式运行 zeroclaw daemon

在任务计划程序 GUI（taskschd.msc）中，依次展开“任务计划程序库” → “ZeroClaw”进行验证。

日志文件位于 %LOCALAPPDATA%\ZeroClaw\logs\：

type %LOCALAPPDATA%\ZeroClaw\logs\zeroclaw.log

Windows 服务（系统范围）

对于服务器或多用户 Windows 安装，请以管理员身份运行 zeroclaw service install：

:: 管理员 cmd.exe
zeroclaw service install

以管理员身份运行会使安装程序在 LocalSystem 下注册一个真实的 Windows 服务，而不是用户范围的计划任务。通过 services.msc 或：

sc query ZeroClaw
sc start ZeroClaw
sc stop ZeroClaw

配置路径解析

该服务从安装时所在的任何工作区读取配置。顺序如下：

  1. 如果设置了 $ZEROCLAW_CONFIG_DIR/config.toml
  2. 如果设置了 $ZEROCLAW_WORKSPACE/.zeroclaw/config.toml
  3. $HOMEBREW_PREFIX/var/zeroclaw/.zeroclaw/config.toml（通过 Homebrew 安装时）
  4. ~/.zeroclaw/config.toml（Linux/macOS）或 %USERPROFILE%\.zeroclaw\config.toml（Windows）

如果您的服务似乎忽略了配置更改，请检查守护进程正在读取的路径：

zeroclaw config list

其输出的前几行显示了它解析到的配置文件路径。

自动更新

该服务不会自动更新。这是有意为之——由您自行决定何时获取新代码。请订阅 GitHub 发布源或 Discord 的 #releases 频道（参见 贡献指南 → 沟通方式）。
