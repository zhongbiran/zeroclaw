SOP 连接与事件合并

本文档描述了外部事件如何触发 SOP 运行。

快速路径

  • MQTT 集成
  • Webhook 集成
  • Cron 集成
  • 安全默认设置
  • 故障排除

1. 概述

ZeroClaw 通过统一的 SOP 调度器（dispatch_sop_event）路由 MQTT/webhook/cron/外围设备事件。

关键行为：

  • 一致的触发器匹配： 所有事件源使用一个匹配器路径。
  • 运行开始审计： 已启动的运行通过 SopAuditLogger 进行持久化。
  • 无头模式安全性： 在非智能体循环上下文中，ExecuteStep 操作会被记录为待处理状态（而非静默执行）。

2. MQTT 集成

2.1 配置

使用 zeroclaw config set channels.mqtt.<field> <value> 配置代理访问权限——这些键会存储在配置文件的 [channels.mqtt] 部分。有关所有字段的详细信息，请参阅 配置参考。use_tls 标志必须与 broker_url 的方案匹配（mqtts:// ⇒ true，mqtt:// ⇒ false）。

2.2 触发器定义

在 SOP.toml 中：

[[triggers]]
type = "mqtt"
topic = "sensors/alert"
condition = "$.severity >= 2"

MQTT 负载被转发到 SOP 事件负载（event.payload），然后在步骤上下文中显示。

3. Webhook 集成

3.1 端点

  • POST /sop/{*rest}：仅 SOP 端点。如果没有匹配的 SOP，则返回 404。无 LLM 回退。
  • POST /webhook：聊天端点。它首先尝试 SOP 分发；如果没有匹配，则回退到正常的 LLM 流程。

路径匹配与配置的 Webhook 触发器路径完全一致。

示例：

  • SOP 中的触发路径：path = "/sop/deploy"
  • 匹配请求：POST /sop/deploy

3.2 授权

当启用配对（默认）时，请提供：

  1. Authorization: Bearer <token>（来自 POST /pair）
  2. 可选的第二层：当配置了 webhook 密钥时，使用 X-Webhook-Secret: <secret>

3.3 幂等性

使用：

X-Idempotency-Key: <唯一键>

默认值：

  • TTL：300秒
  • 重复响应：200 OK，其中 "status": "duplicate"

幂等键按端点命名空间隔离（/webhook 与 /sop/*）。

3.4 示例请求

curl -X POST http://127.0.0.1:42617/sop/deploy \
  -H "Authorization: Bearer <token>" \
  -H "X-Idempotency-Key: $(uuidgen)" \
  -H "Content-Type: application/json" \
  -d '{"message":"deploy-service-a"}'

典型响应：

{
  状态: 已接受,
  "matched_sops": ["部署流水线"],
  "源": "sop_webhook",
  路径: "/sop/deploy"
}

4. Cron 集成

调度程序使用基于窗口的检查来评估缓存的 cron 触发器。

  • 基于窗口： 不会遗漏 (last_check, now] 内的事件。
  • 每个表达式每刻最多触发一次： 如果在一个轮询窗口中有多个触发点，则只执行一次分发。

触发器示例：

[[triggers]]
type = "cron"
expression = "0 0 8 * * *"

Cron 表达式支持 5、6 或 7 个字段。

5. 安全默认设置

功能机制
MQTT 传输mqtts:// + use_tls = true 用于 TLS 传输
Webhook 认证配对承载令牌（默认必需），可选共享密钥标头
速率限制每个客户端在 webhook 路由上的限制（webhook_rate_limit_per_minute，默认值为 60）
幂等性基于标头的去重（X-Idempotency-Key，默认 TTL 300s）
Cron 验证在解析/缓存构建期间，无效的 cron 表达式会失败并关闭

6. 故障排除

症状可能的原因修复
MQTT 连接错误代理 URL/TLS 不匹配验证方案与 TLS 标志配对（mqtt:///false、mqtts:///true）
Webhook 401 未授权缺少 bearer 或无效的密钥重新配对令牌（POST /pair）并验证 X-Webhook-Secret（如果已配置）
/sop/* 返回 404触发路径不匹配确保 SOP.toml 使用精确路径（例如 /sop/deploy）
SOP 已启动，但步骤未执行无活动代理循环的无头触发为 ExecuteStep 运行代理循环，或设计运行以在审批时暂停
Cron 未触发守护进程未运行或表达式无效运行 zeroclaw daemon；检查日志中的 cron 解析警告
