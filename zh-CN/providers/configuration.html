提供程序配置

每个模型提供商都在 ~/.zeroclaw/config.toml 中的 [providers.models.<name>] 下声明。<name> 是你自定义的别名——它用于在配置的其他部分引用该提供商（例如 default_model = "claude"、fallback_providers = ["claude", "local"] 等）。

最小形状

[providers.models.<name>]
kind = "<provider-kind>"   # 必需 — 选择实现方式
model = "<model-id>"       # 必需 — 传递给 provider

几乎所有提供商也接受：

api_key = "..."            # 或环境变量占位符
base_url = "https://..."   # 用于 OpenAI 兼容或自托管端点

凭证

四种提供凭证的方式，按解析顺序排列：

  1. 在配置项中内联 api_key = "..."（适用于开发环境，但对于已提交到代码库的配置存在风险）
  2. 配置级密钥存储 — 通过本地密钥文件在 ~/.zeroclaw/secrets 处进行加密
  3. 特定于提供者的环境变量 — ANTHROPIC_API_KEY、ANTHROPIC_OAUTH_TOKEN、OPENAI_API_KEY、OPENROUTER_API_KEY、GROQ_API_KEY 等。
  4. 通用回退 — ZEROCLAW_API_KEY、API_KEY

默认情况下，入门向导会将凭据写入密钥存储区。您提交的配置文件不应使用内联密钥或会导致用户密钥泄露的 env_passthrough 条目。

OAuth 和订阅认证

多个提供商支持 OAuth / 订阅式令牌，而不是原始 API 密钥：

  • Anthropic — sk-ant-oat-* OAuth 令牌在 API 密钥可以使用的任何地方均可工作。如果您使用的是 Pro/Team 计划，则无需额外费用。
  • GitHub Copilot — 通过入门向导（zeroclaw onboard）进行身份验证，该向导处理 OAuth 流程。令牌存储在密钥后端中。
  • Gemini CLI — 使用 gemini CLI 的现有认证机制。
  • Claude Code — 使用您的 Claude Code 登录信息。

容器友好的覆盖

引导向导会检测 Docker/Podman/Kubernetes，并将 localhost 重写为适合容器的主机名：

[providers.models.local]
kind = "ollama"
base_url = "http://host.docker.internal:11434"   # 在宿主机上为 "http://localhost:11434"

你也可以在运行时手动强制执行此操作：

ZEROCLAW_OLLAMA_BASE_URL=http://ollama:11434 zeroclaw agent

每个提供者的配置选项

除了通用字段外，某些提供商还接受额外字段。亮点如下：

Ollama

[providers.models.local]
kind = "ollama"
base_url = "http://localhost:11434"
model = "qwen3.6:35b-a3b"
think = false                # 禁用推理模式以加快输出速度
reasoning_effort = "none"    # 相同意图，作为顶层字段传递
options = { temperature = 0, num_ctx = 32768 }

与 OpenAI 兼容

[providers.models.groq]
kind = "openai-compatible"
base_url = "https://api.groq.com/openai"
model = "llama-3.3-70b-versatile"
api_key = "gsk_..."
# 可选 — 提供端点支持的 SSE 工具调用流式提示
native_tool_streaming = true

Azure OpenAI

[providers.models.azure]
kind = "azure-openai"
base_url = "https://my-resource.openai.azure.com"
deployment = "gpt-4o"
api_version = "2024-10-01-preview"
api_key = "..."

选择默认值

default_provider = "claude"
default_model    = "claude-haiku-4-5-20251001"

两者均在代理启动时读取。频道、工具和标准操作程序（SOP）可以针对每个请求进行覆盖。
