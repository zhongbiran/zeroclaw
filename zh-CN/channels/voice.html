语音与电话

实时语音输入和输出。四个通道覆盖矩阵：入站呼叫、出站语音合成、本地麦克风唤醒以及 SIP 级实时对话。

ClawdTalk（实时 SIP）

[channels.clawdtalk]
enabled = true
telnyx_api_key = "..."
connection_id = "..."              # Telnyx SIP 连接
model_provider = "voice-brain"     # 引用 [providers.models.voice-brain]
voice = "Polly.Joanna-Neural"      # Telnyx 语音 ID

由 Telnyx 提供支持的全双工 SIP 语音。代理通过真实电话通话（入站或出站）进行交流。支持打断、中途工具调用和区域号码配置。

配对： 为大脑（crates/zeroclaw-providers/src/telnyx.rs）使用 telnyx 提供程序，并确保你的 Telnyx 账户中有一个 SIP 连接，其 webhook URL 指向 ZeroClaw 网关。

语音通话（Twilio / Telnyx / Plivo）

[channels.voice_call]
enabled = true
carrier = "twilio"                 # 或 "telnyx", "plivo"
account_sid = "..."
auth_token = "..."
from_number = "+14155550123"
stt_provider = "whisper"
tts_provider = "elevenlabs"

传统运营商语音——代理接听，通过 STT 进行转录，再通过 TTS 回复。延迟高于 ClawdTalk，但适用于任何普通电话号码，且无需 SIP 中继配置。

语音唤醒（本地唤醒词）

[channels.voice_wake]
enabled = true
wake_phrase = "hey claw"
engine = "porcupine"               # 或 "openwakeword"
model_path = "~/.zeroclaw/wake/hey-claw.ppn"
audio_device = "default"

在本地运行，监听麦克风，在听到唤醒词时触发代理交互。适用于：

  • SBC 上的物理语音助手
  • 桌面端“热词 → 提问”工作流
  • 始终在线的家庭自动化代理

代理不会将音频发送到任何地方——唤醒检测是本地进行的。只有唤醒后的语音会通过 STT 并到达 LLM。

TTS（出站语音合成）

[channels.tts]
enabled = true
engine = "piper"                   # 本地，免费
voice = "en_US-lessac-medium"
output_device = "default"

其他引擎：

[channels.tts]
engine = "openai"                  # TTS-1 或 TTS-1-HD
voice = "alloy"
api_key = "..."

[channels.tts]
engine = "elevenlabs"
voice_id = "21m00Tcm4TlvDq8ikWAM"
api_key = "..."

[channels.tts]
engine = "google-cloud"
voice = "en-US-Neural2-J"
credentials_json = "~/.zeroclaw/gcp.json"

[channels.tts]
engine = "edge"                    # Microsoft Edge TTS — 免费，在线
voice = "en-US-AndrewNeural"

TTS 仅用于输出——它不是对话通道，而是其他通道回复的扬声器。与 voice_wake 配合使用，即可构建完整的本地语音助手。

延迟预算

端到端延迟低于约 500 毫秒时，语音听起来是实时的。实际预算：

组件典型延迟
唤醒检测（本地）<100 毫秒
STT（Whisper 本地）每条话语 300–800 毫秒
LLM 首字100–2000 毫秒（取决于模型）
TTS 首次音频200–700 毫秒
网络（蜂窝 / PSTN）100–300 毫秒往返时间（RTT）

ClawdTalk 通过保持音频流实时传输，简化了其中几个步骤；而常规的 voice_call 则依次经历 STT + LLM + TTS。

STT 选项

运行时根据 stt_provider 选择 STT：

  • whisper — 本地 whisper.cpp；仅 CPU 可用，但 GPU 能提升性能
  • openai-whisper — 托管的 Whisper API
  • deepgram — 实时流式语音转文本（最快）
  • google-cloud-stt
  • azure-stt

设置在语音频道的配置中（stt_provider = "deepgram"）。

硬件说明

对于始终开启的语音功能，在 SBC 上：

  • USB 麦克风：任何符合 UAC 标准的麦克风均可使用。运行 arecord -l 以验证操作系统是否已识别该设备。
  • 扬声器：可以是 USB 音频输出或 SBC 板载插孔；将 output_device 设置为 ALSA 名称
  • 内置 AEC（声学回声消除）功能的麦克风在扬声器靠近时能显著提高唤醒可靠性

有关 Android 特定的音频设置，请参阅 硬件 → Android。
