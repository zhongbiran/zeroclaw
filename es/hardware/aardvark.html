Capa por capa

Capa 1 — aardvark-sys (el hablante USB)

Archivo: crates/aardvark-sys/src/lib.rs

Esta es la única capa que interactúa directamente con la biblioteca C en bruto. Piensa en ella como un traductor ligero: convierte las llamadas a funciones C en Rust seguro.

Algoritmo:

find_devices()
  → llamar aa_find_devices(16, buf)       // preguntar a la lib C cuántos adaptadores hay
  → devolver Vec de números de puerto     // [0, 1, ...] uno por adaptador

open_port(port)
  → llamar aa_open(port)                  // abrir ese adaptador específico
  → si handle ≤ 0, devolver OpenFailed
  → de lo contrario devolver AardvarkHandle{ _port: handle }

i2c_scan(handle)
  → para addr en 0x08..=0x77            // cada dirección de 7 bits válida
      intentar aa_i2c_read(addr, 1 byte) // llamar a la puerta
      si ACK → añadir a la lista          // el dispositivo respondió
  → devolver lista de direcciones activas

i2c_read(handle, addr, len)
  → aa_i2c_read(addr, len bytes)
  → devolver bytes como Vec<u8>

i2c_write(handle, addr, data)
  → aa_i2c_write(addr, data)

spi_transfer(handle, bytes_to_send)
  → aa_spi_write(bytes)                // full-duplex: envía + recibe
  → devolver bytes recibidos

gpio_set(handle, direction, value)
  → aa_gpio_direction(direction)       // qué pines son salidas
  → aa_gpio_put(value)                 // establecer niveles de salida

gpio_get(handle)
  → aa_gpio_get()                      // leer todos los niveles de pines como bitmask

Drop(handle)
  → aa_close(handle._port)            // siempre cerrar al eliminar

En modo stub (sin SDK): cada método devuelve Err(NotFound) de inmediato. find_devices() devuelve []. Nada se bloquea.

Capa 2 — AardvarkTransport (el puente)

Archivo: crates/zeroclaw-hardware/src/aardvark.rs

El resto de ZeroClaw habla un único idioma: ZcCommand → ZcResponse. AardvarkTransport traduce entre ese protocolo y las llamadas de aardvark-sys anteriores.

Algoritmo:

send(ZcCommand) → ZcResponse

  extraer el nombre del comando desde cmd.name
  extraer los parámetros desde cmd.params (valores serde_json)

  coincidir cmd.name:

    "i2c_scan"   → abrir el manejador → llamar a i2c_scan()
                   → formatear las direcciones encontradas como lista hexadecimal
                   → devolver ZcResponse{ output: "0x48, 0x68" }

    "i2c_read"   → analizar addr (cadena hexadecimal) + len (número)
                   → abrir el manejador → i2c_enable(bitrate)
                   → llamar a i2c_read(addr, len)
                   → formatear los bytes como hexadecimal
                   → devolver ZcResponse{ output: "0xAB 0xCD" }

    "i2c_write"  → analizar addr + bytes de datos
                   → abrir el manejador → i2c_write(addr, data)
                   → devolver ZcResponse{ output: "ok" }

    "spi_transfer" → analizar bytes_hex (cadena) → decodificar a Vec<u8>
                     → abrir el manejador → spi_enable(bitrate)
                     → spi_transfer(bytes)
                     → devolver los bytes recibidos como hexadecimal

    "gpio_set"   → analizar máscaras de bits de dirección + valor
                   → abrir el manejador → gpio_set(dir, val)
                   → devolver ZcResponse{ output: "ok" }

    "gpio_get"   → abrir el manejador → gpio_get()
                   → devolver el valor de la máscara de bits como cadena

  en caso de cualquier AardvarkError → devolver ZcResponse{ error: "..." }

Elección de diseño clave — apertura diferida: El manejador se abre desde cero para cada comando y se cierra al final. Esto significa que no hay conexión mantenida, ningún estado que limpiar y ninguna lógica de “¿sigue abierto?” en ningún lugar.

Capa 3 — Herramientas (lo que el agente llama)

Archivo: crates/zeroclaw-hardware/src/aardvark_tools.rs

Cada herramienta es un envoltorio delgado. Este:

  1. Valida la entrada JSON del agente
  2. Resuelve qué dispositivo físico utilizar
  3. Construye un ZcCommand
  4. Llama a AardvarkTransport.send()
  5. Devuelve el resultado como texto
I2cScanTool.call(args)
  → buscar "device" en args (predeterminado: "aardvark0")
  → encontrar ese dispositivo en el registro
  → construir ZcCommand{ name: "i2c_scan", params: {} }
  → enviar a AardvarkTransport
  → devolver "Found: 0x48, 0x68" (o "No devices found")

I2cReadTool.call(args)
  → requerir args["addr"] y args["len"]
  → construir ZcCommand{ name: "i2c_read", params: {addr, len} }
  → enviar → devolver bytes en hexadecimal

I2cWriteTool.call(args)
  → requerir args["addr"] y args["data"] (hex o array)
  → construir ZcCommand{ name: "i2c_write", params: {addr, data} }
  → enviar → devolver "ok" o error

SpiTransferTool.call(args)
  → requerir args["bytes"] (cadena hexadecimal)
  → construir ZcCommand{ name: "spi_transfer", params: {bytes} }
  → enviar → devolver bytes recibidos

GpioAardvarkTool.call(args)
  → requerir args["direction"] + args["value"]  (set)
         O sin argumentos adicionales               (get)
  → construir ZcCommand apropiado
  → enviar → devolver resultado

DatasheetTool.call(args)
  → action = args["action"]: "search" | "download" | "list" | "read"
  → "search":   devolver una URL de búsqueda de Google/fabricante para el dispositivo
  → "download": obtener PDF desde args["url"] → guardar en ~/.zeroclaw/hardware/datasheets/
  → "list":     escanear el directorio de datasheets → devolver nombres de archivo
  → "read":     abrir un PDF guardado y devolver su texto

Capa 4 — Registro de dispositivos (la libreta de direcciones)

Archivo: crates/zeroclaw-hardware/src/device.rs

El registro es un mapa en tiempo de ejecución de todos los dispositivos conectados. Cada entrada almacena: alias, tipo, capacidades y controlador de transporte.

register("aardvark", vid=0x2b76, ...)
  → DeviceKind::from_vid(0x2b76)  → DeviceKind::Aardvark
  → DeviceRuntime::from_kind()    → DeviceRuntime::Aardvark
  → asignar alias "aardvark0" (luego "aardvark1" para el segundo, etc.)
  → almacenar entrada en HashMap

attach_transport("aardvark0", AardvarkTransport, capacidades{i2c,spi,gpio})
  → almacenar Arc<dyn Transport> en la entrada

has_aardvark()
  → cualquier entrada donde kind == Aardvark  → true / false

resolve_aardvark_device(args)
  → leer parámetro "device" (predeterminado: "aardvark0")
  → buscar alias en HashMap
  → devolver (alias, DeviceContext{ transport, capacidades })

Capa 5 — boot() (configuración de inicio)

Archivo: crates/zeroclaw-hardware/src/lib.rs

boot() se ejecuta una vez al inicio. Para Aardvark:

boot()
  ...
  aardvark_ports = aardvark_sys::AardvarkHandle::find_devices()
  // → [] en modo stub, [0] si hay un adaptador conectado

  for (i, port) in aardvark_ports:
    alias = registry.register("aardvark", vid=0x2b76, ...)
    // → "aardvark0", "aardvark1", ...

    transport = AardvarkTransport::new(port, bitrate=100kHz)
    registry.attach_transport(alias, transport, {i2c:true, spi:true, gpio:true})

    log "[registry] aardvark0 listo → Puerto 0 de Total Phase"
  ...

Capa 6 — Registro de herramientas (el cargador)

Archivo: crates/zeroclaw-hardware/src/tool_registry.rs

Después de boot(), el registro de herramientas verifica qué hardware está presente y carga solo las herramientas relevantes:

ToolRegistry::load(devices)

  # always loaded (Pico / GPIO)
  register: gpio_write, gpio_read, gpio_toggle, pico_flash, device_list, device_status

  # only loaded if an Aardvark was found at boot
  if devices.has_aardvark():
    register: i2c_scan, i2c_read, i2c_write, spi_transfer, gpio_aardvark, datasheet

Por eso la prueba hardware_feature_registers_all_six_tools sigue pasando en modo stub: has_aardvark() devuelve false, no se cargan herramientas adicionales y el contador se mantiene en 6.

 MENSAJE DEL USUARIO: "escanear el bus i2c"

  bucle del agente
      │
      ▼
  I2cScanTool.call()
      │
      ▼
  resolve_aardvark_device("aardvark0")
      │  devuelve el Arc del transporte
      ▼
  AardvarkTransport.send(ZcCommand{ name: "i2c_scan" })
      │
      ▼
  AardvarkHandle::open_port(0)    ← abre la conexión USB
      │
      ▼
  aa_i2c_read(0x08..0x77)         ← sondea cada dirección
      │
      ▼
  AardvarkHandle dropped           ← cierra la conexión USB
      │
      ▼
  ZcResponse{ output: "Encontrado: 0x48, 0x68" }
      │
      ▼
  el agente envía la respuesta al usuario: "Encontré dos dispositivos I2C: 0x48 y 0x68"
